11//! This crate is an extension to the popular [gotham web framework][gotham] for Rust. It allows you to
12//! create resources with assigned endpoints that aim to be a more convenient way of creating handlers
13//! for requests.
14//!
15//! # Features
16//!
17//!  - Automatically parse **JSON** request and produce response bodies
18//!  - Allow using **raw** request and response bodies
19//!  - Convenient **macros** to create responses that can be registered with gotham's router
20//!  - Auto-Generate an **OpenAPI** specification for your API
21//!  - Manage **CORS** headers so you don't have to
22//!  - Manage **Authentication** with JWT
23//!  - Integrate diesel connection pools for easy **database** integration
24//!
25//! # Safety
26//!
27//! This crate is just as safe as you'd expect from anything written in safe Rust - and
28//! `#![forbid(unsafe_code)]` ensures that no unsafe was used.
29//!
30//! # Endpoints
31//!
32//! There are a set of pre-defined endpoints that should cover the majority of REST APIs. However,
33//! it is also possible to define your own endpoints.
34//!
35//! ## Pre-defined Endpoints
36//!
37//! Assuming you assign `/foobar` to your resource, the following pre-defined endpoints exist:
38//!
39//! | Endpoint Name | Required Arguments | HTTP Verb | HTTP Path      |
40//! | ------------- | ------------------ | --------- | -------------- |
41//! | read_all      |                    | GET       | /foobar        |
42//! | read          | id                 | GET       | /foobar/:id    |
43//! | search        | query              | GET       | /foobar/search |
44//! | create        | body               | POST      | /foobar        |
45//! | update_all    | body               | PUT       | /foobar        |
46//! | update        | id, body           | PUT       | /foobar/:id    |
47//! | delete_all    |                    | DELETE    | /foobar        |
48//! | delete        | id                 | DELETE    | /foobar/:id    |
49//!
50//! Each of those endpoints has a macro that creates the neccessary boilerplate for the Resource. A
51//! simple example looks like this:
52//!
53//! ```rust,no_run
54//! # #[macro_use] extern crate gotham_restful_derive;
55//! # use gotham::router::builder::*;
56//! # use gotham_restful::*;
57//! # use serde::{Deserialize, Serialize};
58//! /// Our RESTful resource.
59//! #[derive(Resource)]
60//! #[resource(read)]
61//! struct FooResource;
62//!
63//! /// The return type of the foo read endpoint.
64//! #[derive(Serialize)]
65//! # #[cfg_attr(feature = "openapi", derive(openapi_type::OpenapiType))]
66//! struct Foo {
67//! 	id: u64
68//! }
69//!
70//! /// The foo read endpoint.
71//! #[read]
72//! fn read(id: u64) -> Success<Foo> {
73//! 	Foo { id }.into()
74//! }
75//! # fn main() {
76//! # 	gotham::start("127.0.0.1:8080", build_simple_router(|route| {
77//! # 		route.resource::<FooResource>("foo");
78//! # 	})).expect("Failed to start gotham");
79//! # }
80//! ```
81//!
82//! ## Custom Endpoints
83//!
84//! Defining custom endpoints is done with the `#[endpoint]` macro. The syntax is similar to that
85//! of the pre-defined endpoints, but you need to give it more context:
86//!
87//! ```rust,no_run
88//! # #[macro_use] extern crate gotham_restful_derive;
89//! # use gotham::{router::build_simple_router, prelude::*};
90//! # use gotham_restful::*;
91//! # use serde::{Deserialize, Serialize};
92//! use gotham_restful::gotham::hyper::Method;
93//!
94//! #[derive(Resource)]
95//! #[resource(custom_endpoint)]
96//! struct CustomResource;
97//!
98//! /// This type is used to parse path parameters.
99//! #[derive(Clone, Deserialize, StateData, StaticResponseExtender)]
100//! # #[cfg_attr(feature = "openapi", derive(openapi_type::OpenapiType))]
101//! struct CustomPath {
102//! 	name: String
103//! }
104//!
105//! #[endpoint(
106//! 	uri = "custom/:name/read",
107//! 	method = "Method::GET",
108//! 	params = false,
109//! 	body = false
110//! )]
111//! fn custom_endpoint(path: CustomPath) -> Success<String> {
112//! 	path.name.into()
113//! }
114//! # fn main() {
115//! # 	gotham::start("127.0.0.1:8080", build_simple_router(|route| {
116//! # 		route.resource::<CustomResource>("custom");
117//! # 	})).expect("Failed to start gotham");
118//! # }
119//! ```
120//!
121//! # Arguments
122//!
123//! Some endpoints require arguments. Those should be
124//!  * **id** Should be a deserializable json-primitive like [`i64`] or [`String`].
125//!  * **body** Should be any deserializable object, or any type implementing [`RequestBody`].
126//!  * **query** Should be any deserializable object whose variables are json-primitives. It will
127//!    however not be parsed from json, but from HTTP GET parameters like in `search?id=1`. The
128//!    type needs to implement [`QueryStringExtractor`](gotham::extractor::QueryStringExtractor).
129//!
130//! Additionally, all handlers may take a reference to gotham's [`State`]. Please note that for async
131//! handlers, it needs to be a mutable reference until rustc's lifetime checks across await bounds
132//! improve.
133//!
134//! # Uploads and Downloads
135//!
136//! By default, every request body is parsed from json, and every respone is converted to json using
137//! [serde_json]. However, you may also use raw bodies. This is an example where the request body
138//! is simply returned as the response again, no json parsing involved:
139//!
140//! ```rust,no_run
141//! # #[macro_use] extern crate gotham_restful_derive;
142//! # use gotham::{mime::{self, Mime}, router::builder::*};
143//! # use gotham_restful::*;
144//! # use serde::{Deserialize, Serialize};
145//! #[derive(Resource)]
146//! #[resource(create)]
147//! struct ImageResource;
148//!
149//! #[derive(FromBody, RequestBody)]
150//! #[supported_types(mime::IMAGE_GIF, mime::IMAGE_JPEG, mime::IMAGE_PNG)]
151//! struct RawImage {
152//! 	content: Vec<u8>,
153//! 	content_type: Mime
154//! }
155//!
156//! #[create]
157//! fn create(body: RawImage) -> Raw<Vec<u8>> {
158//! 	Raw::new(body.content, body.content_type)
159//! }
160//! # fn main() {
161//! # 	gotham::start("127.0.0.1:8080", build_simple_router(|route| {
162//! # 		route.resource::<ImageResource>("image");
163//! # 	})).expect("Failed to start gotham");
164//! # }
165//! ```
166//!
167//! # Custom HTTP Headers
168//!
169//! You can read request headers from the state as you would in any other gotham handler, and specify
170//! custom response headers using [Response::header].
171//!
172//! ```rust,no_run
173//! # #[macro_use] extern crate gotham_restful_derive;
174//! # use gotham::hyper::header::{ACCEPT, HeaderMap, VARY};
175//! # use gotham::{router::builder::*, state::State};
176//! # use gotham_restful::*;
177//! #[derive(Resource)]
178//! #[resource(read_all)]
179//! struct FooResource;
180//!
181//! #[read_all]
182//! async fn read_all(state: &mut State) -> NoContent {
183//! 	let headers: &HeaderMap = state.borrow();
184//! 	let accept = &headers[ACCEPT];
185//! # drop(accept);
186//!
187//! 	let mut res = NoContent::default();
188//! 	res.header(VARY, "accept".parse().unwrap());
189//! 	res
190//! }
191//! # fn main() {
192//! # 	gotham::start("127.0.0.1:8080", build_simple_router(|route| {
193//! # 		route.resource::<FooResource>("foo");
194//! # 	})).expect("Failed to start gotham");
195//! # }
196//! ```
197//!
198//! # Features
199//!
200//! To make life easier for common use-cases, this create offers a few features that might be helpful
201//! when you implement your web server.  The complete feature list is
202//!  - [`auth`](#authentication-feature) Advanced JWT middleware
203//!  - [`cors`](#cors-feature) CORS handling for all endpoint handlers
204//!  - [`database`](#database-feature) diesel middleware support
205//!  - `errorlog` log errors returned from endpoint handlers
206//!  - `full` enables all features except `without-openapi`
207//!  - [`openapi`](#openapi-feature) router additions to generate an openapi spec
208//!  - `without-openapi` (**default**) disables `openapi` support.
209//!
210//! ## Authentication Feature
211//!
212//! In order to enable authentication support, enable the `auth` feature gate. This allows you to
213//! register a middleware that can automatically check for the existence of an JWT authentication
214//! token. Besides being supported by the endpoint macros, it supports to lookup the required JWT secret
215//! with the JWT data, hence you can use several JWT secrets and decide on the fly which secret to use.
216//! None of this is currently supported by gotham's own JWT middleware.
217//!
218//! A simple example that uses only a single secret looks like this:
219//!
220//! ```rust,no_run
221//! # #[macro_use] extern crate gotham_restful_derive;
222//! # #[cfg(feature = "auth")]
223//! # mod auth_feature_enabled {
224//! # use gotham::{router::builder::*, pipeline::*, state::State};
225//! # use gotham_restful::*;
226//! # use serde::{Deserialize, Serialize};
227//! #[derive(Resource)]
228//! #[resource(read)]
229//! struct SecretResource;
230//!
231//! #[derive(Serialize)]
232//! # #[cfg_attr(feature = "openapi", derive(openapi_type::OpenapiType))]
233//! struct Secret {
234//! 	id: u64,
235//! 	intended_for: String
236//! }
237//!
238//! #[derive(Deserialize, Clone)]
239//! struct AuthData {
240//! 	sub: String,
241//! 	exp: u64
242//! }
243//!
244//! #[read]
245//! fn read(auth: AuthStatus<AuthData>, id: u64) -> AuthSuccess<Secret> {
246//! 	let intended_for = auth.ok()?.sub;
247//! 	Ok(Secret { id, intended_for })
248//! }
249//!
250//! fn main() {
251//! 	let auth: AuthMiddleware<AuthData, _> = AuthMiddleware::new(
252//! 		AuthSource::AuthorizationHeader,
253//! 		AuthValidation::default(),
254//! 		StaticAuthHandler::from_array(b"zlBsA2QXnkmpe0QTh8uCvtAEa4j33YAc")
255//! 	);
256//! 	let (chain, pipelines) = single_pipeline(new_pipeline().add(auth).build());
257//! 	gotham::start(
258//! 		"127.0.0.1:8080",
259//! 		build_router(chain, pipelines, |route| {
260//! 			route.resource::<SecretResource>("secret");
261//! 		})
262//! 	)
263//! 	.expect("Failed to start gotham");
264//! }
265//! # }
266//! ```

268//! ## CORS Feature
269//!
270//! The cors feature allows an easy usage of this web server from other origins. By default, only
271//! the `Access-Control-Allow-Methods` header is touched. To change the behaviour, add your desired
272//! configuration as a middleware.
273//!
274//! A simple example that allows authentication from every origin (note that `*` always disallows
275//! authentication), and every content type, looks like this:
276//!
277//! ```rust,no_run
278//! # #[macro_use] extern crate gotham_restful_derive;
279//! # #[cfg(feature = "cors")]
280//! # mod cors_feature_enabled {
281//! # use gotham::{hyper::header::*, router::builder::*, pipeline::*, state::State};
282//! # use gotham_restful::{*, cors::*};
283//! # use serde::{Deserialize, Serialize};
284//! #[derive(Resource)]
285//! #[resource(read_all)]
286//! struct FooResource;
287//!
288//! #[read_all]
289//! fn read_all() {
290//! 	// your handler
291//! }
292//!
293//! fn main() {
294//! 	let cors = CorsConfig {
295//! 		origin: Origin::Copy,
296//! 		headers: Headers::List(vec![CONTENT_TYPE]),
297//! 		max_age: 0,
298//! 		credentials: true
299//! 	};
300//! 	let (chain, pipelines) = single_pipeline(new_pipeline().add(cors).build());
301//! 	gotham::start(
302//! 		"127.0.0.1:8080",
303//! 		build_router(chain, pipelines, |route| {
304//! 			route.resource::<FooResource>("foo");
305//! 		})
306//! 	)
307//! 	.expect("Failed to start gotham");
308//! }
309//! # }
310//! ```
311//!
312//! The cors feature can also be used for non-resource handlers. Take a look at [`CorsRoute`]
313//! for an example.
314//!
315//! ## Database Feature
316//!
317//! The database feature allows an easy integration of [diesel] into your handler functions. Please
318//! note however that due to the way gotham's diesel middleware implementation, it is not possible
319//! to run async code while holding a database connection. If you need to combine async and database,
320//! you'll need to borrow the connection from the [`State`] yourself and return a boxed future.
321//!
322//! A simple non-async example looks like this:
323//!
324//! ```rust,no_run
325//! # #[macro_use] extern crate diesel;
326//! # #[macro_use] extern crate gotham_restful_derive;
327//! # #[cfg(feature = "database")]
328//! # mod database_feature_enabled {
329//! # use diesel::{table, PgConnection, QueryResult, RunQueryDsl};
330//! # use gotham::{router::builder::*, pipeline::*, state::State};
331//! # use gotham_middleware_diesel::DieselMiddleware;
332//! # use gotham_restful::*;
333//! # use serde::{Deserialize, Serialize};
334//! # use std::env;
335//! # table! {
336//! #   foo (id) {
337//! #     id -> Int8,
338//! #     value -> Text,
339//! #   }
340//! # }
341//! #[derive(Resource)]
342//! #[resource(read_all)]
343//! struct FooResource;
344//!
345//! #[derive(Queryable, Serialize)]
346//! # #[cfg_attr(feature = "openapi", derive(openapi_type::OpenapiType))]
347//! struct Foo {
348//! 	id: i64,
349//! 	value: String
350//! }
351//!
352//! #[read_all]
353//! fn read_all(conn: &mut PgConnection) -> QueryResult<Vec<Foo>> {
354//! 	foo::table.load(conn)
355//! }
356//!
357//! type Repo = gotham_middleware_diesel::Repo<PgConnection>;
358//!
359//! fn main() {
360//! 	let repo = Repo::new(&env::var("DATABASE_URL").unwrap());
361//! 	let diesel = DieselMiddleware::new(repo);
362//!
363//! 	let (chain, pipelines) = single_pipeline(new_pipeline().add(diesel).build());
364//! 	gotham::start(
365//! 		"127.0.0.1:8080",
366//! 		build_router(chain, pipelines, |route| {
367//! 			route.resource::<FooResource>("foo");
368//! 		})
369//! 	)
370//! 	.expect("Failed to start gotham");
371//! }
372//! # }
373//! ```

375//! ## OpenAPI Feature
376//!
377//! The OpenAPI feature is probably the most powerful one of this crate. Definitely read this section
378//! carefully both as a binary as well as a library author to avoid unwanted suprises.
379//!
380//! In order to automatically create an openapi specification, gotham-restful needs knowledge over
381//! all routes and the types returned. `serde` does a great job at serialization but doesn't give
382//! enough type information, so all types used in the router need to implement
383//! [`OpenapiType`](openapi_type::OpenapiType). This can be derived for almoust any type and there
384//! should be no need to implement it manually. A simple example looks like this:
385//!
386//! ```rust,no_run
387//! # #[macro_use] extern crate gotham_restful_derive;
388//! # #[cfg(feature = "openapi")]
389//! # mod openapi_feature_enabled {
390//! # use gotham::{router::builder::*, state::State};
391//! # use gotham_restful::*;
392//! # use openapi_type::OpenapiType;
393//! # use serde::{Deserialize, Serialize};
394//! #[derive(Resource)]
395//! #[resource(read_all)]
396//! struct FooResource;
397//!
398//! #[derive(OpenapiType, Serialize)]
399//! struct Foo {
400//! 	bar: String
401//! }
402//!
403//! #[read_all]
404//! fn read_all() -> Success<Foo> {
405//! 	Foo {
406//! 		bar: "Hello World".to_owned()
407//! 	}
408//! 	.into()
409//! }
410//!
411//! fn main() {
412//! 	gotham::start(
413//! 		"127.0.0.1:8080",
414//! 		build_simple_router(|route| {
415//! 			let info = OpenapiInfo {
416//! 				title: "My Foo API".to_owned(),
417//! 				version: "0.1.0".to_owned(),
418//! 				urls: vec!["https://example.org/foo/api/v1".to_owned()]
419//! 			};
420//! 			route.with_openapi(info, |mut route| {
421//! 				route.resource::<FooResource>("foo");
422//! 				route.openapi_spec("openapi");
423//! 				route.openapi_doc("/");
424//! 			});
425//! 		})
426//! 	)
427//! 	.expect("Failed to start gotham");
428//! }
429//! # }
430//! ```
431//!
432//! Above example adds the resource as before, but adds two other endpoints as well: `/openapi` and `/`.
433//! The first one will return the generated openapi specification in JSON format, allowing you to easily
434//! generate clients in different languages without worying to exactly replicate your api in each of those
435//! languages. The second one will return documentation in HTML format, so you can easily view your
436//! api and share it with other people.
437//!
438//! ### Gotchas
439//!
440//! The openapi feature has some gotchas you should be aware of.
441//!
442//!   - The name of a struct is used as a "link" in the openapi specification. Therefore, if you have two
443//!     structs with the same name in your project, the openapi specification will be invalid as only one
444//!     of the two will make it into the spec.
445//!   - By default, the `without-openapi` feature of this crate is enabled. Disabling it in favour of the
446//!     `openapi` feature will add additional type bounds and method requirements to some of the traits and
447//! 	types in this crate, for example instead of [`Endpoint`] you now have to implement
448//! 	[`EndpointWithSchema`]. This means that some code might only compile on either feature, but not
449//!     on both. If you are writing a library that uses gotham-restful, it is strongly recommended to pass
450//! 	both features through and conditionally enable the openapi code, like this:
451//!
452//!     ```rust
453//!     # #[macro_use] extern crate gotham_restful;
454//!     # use serde::{Deserialize, Serialize};
455//!     #[derive(Deserialize, Serialize)]
456//!     #[cfg_attr(feature = "openapi", derive(openapi_type::OpenapiType))]
457//!     struct Foo;
458//!     ```
459//!
460//!  [diesel]: https://diesel.rs/
461//!  [`State`]: gotham::state::State
462
463#[cfg(all(feature = "openapi", feature = "without-openapi"))]
464compile_error!("The 'openapi' and 'without-openapi' features cannot be combined");
465
466#[cfg(all(not(feature = "openapi"), not(feature = "without-openapi")))]
467compile_error!("Either the 'openapi' or 'without-openapi' feature needs to be enabled");

590/// This trait must be implemented for every resource. It allows you to register the different
591/// endpoints that can be handled by this resource to be registered with the underlying router.
592///
593/// It is not recommended to implement this yourself, just use `#[derive(Resource)]`.

596	/// Register all methods handled by this resource with the underlying router.
597	#[openapi_bound(D: crate::DrawResourceRoutesWithSchema)]
598	#[non_openapi_bound(D: crate::DrawResourceRoutes)]
599	fn setup<D>(route: D);
